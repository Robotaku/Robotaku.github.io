# Googleログイン設定ガイド

このアプリでGoogleアカウントログイン＋Google Drive保存を有効にするための手順です。
**この設定をしないと、ログインボタンを押しても必ずエラーになります。**

---

## 本プログラムの実行方法の注意

**`index.html` をダブルクリックして `file://...` として開いている場合、Googleログインの仕組み（Google Identity Services）はセキュリティ上の理由で一切動作しません。** 必ず `http://` または `https://` 経由でアプリを開く必要があります（下記の手順4参照）。

今回のコード修正で、上記2つの状態を検知して、画面上にわかりやすいエラーメッセージを表示するようにしました。まずは実際にどちらのエラーが出ているか、ログイン画面の赤い注意書きで確認してください。

---

## 手順1: Google Cloud プロジェクトを作成する

1. https://console.cloud.google.com/ にアクセスし、Googleアカウントでログイン
2. 画面上部のプロジェクト選択メニューから「新しいプロジェクト」
3. 適当な名前（例: `toeic600-trainer`）を付けて作成

## 手順2: Google Drive APIを有効にする

1. 左メニュー「APIとサービス」→「ライブラリ」
2. 検索窓で「Google Drive API」を検索
3. 「有効にする」をクリック

## 手順3: OAuth同意画面を設定する

1. 「APIとサービス」→「OAuth同意画面」
2. User Type は個人利用なら「外部」を選択
3. アプリ名（例: TOEIC 600 Trainer）、ユーザーサポートメール、デベロッパーの連絡先メールを入力して保存
4. 「スコープ」の追加は不要（コード側で `drive.appdata` を指定しています）
5. 「テストユーザー」に、ログインに使う自分のGoogleアカウントのメールアドレスを追加
   - ※「公開」申請をしない限り、ここに追加したアカウントしかログインできません。個人利用なら「テスト」モードのままで問題ありません

## 手順4: アプリの公開URL（オリジン）を決める

Googleは「どのURLからのログイン要求か」を厳密にチェックします。まず、このアプリをどう配信するか決めてください。

**選択肢A: ローカルで動作確認する場合**

ターミナルでアプリのフォルダに移動し、簡易サーバーを起動します。

```bash
cd toeic-app
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開きます（`file://` ではなくこちらを使うこと）。

**選択肢B: 実際にネット上に公開する場合**

Netlify、Vercel、GitHub Pagesなどにこのフォルダをアップロードし、発行されたURL（例: `https://your-app.netlify.app`）を使います。

どちらの方法を使うにせよ、**そのURL（オリジン）を手順5で正確に登録する必要があります。**

## 手順5: OAuthクライアントIDを作成する

1. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
2. アプリケーションの種類は「ウェブ アプリケーション」を選択
3. 「承認済みの JavaScript 生成元」に、手順4で決めたURLを**そのまま**追加
   - ローカルの場合: `http://localhost:8000`
   - 公開する場合: `https://your-app.netlify.app`
   - ※末尾にスラッシュ `/` を付けない、パス部分（`/index.html`など）は含めない、httpとhttpsを間違えない、これらを1文字でも間違えると「origin is not allowed」エラーになります
4. 「作成」をクリックすると、`〇〇〇〇.apps.googleusercontent.com` の形式のクライアントIDが発行されます

## 手順6: クライアントIDをコードに貼り付ける

`app.js` の以下の行を、手順5で発行された実際の値に書き換えます。

```js
const GOOGLE_CLIENT_ID = 'ここに発行されたクライアントIDを貼り付け.apps.googleusercontent.com';
```

保存したら、手順4で開いたURL（`http://localhost:8000` など）をブラウザで再読み込みしてください。

## よくあるエラーと対処法

| 症状・メッセージ | 原因 | 対処 |
|---|---|---|
| ログイン画面に「Client IDが未設定」の赤い警告が出る | プレースホルダーのまま | 手順5・6を実施 |
| ログイン画面に「file://で開かれています」の警告が出る | HTMLを直接ダブルクリックして開いている | 手順4のローカルサーバー経由で開き直す |
| ポップアップに `Error 400: redirect_uri_mismatch` または `origin_mismatch` | 承認済みJavaScript生成元と実際のURLが不一致 | 手順5を見直し、プロトコル(http/https)・ドメイン・ポート番号が完全一致しているか確認 |
| ポップアップに `Error 401: invalid_client` | クライアントIDが間違っている、または削除された | 手順5で新しいIDを再発行し、手順6で貼り直す |
| `Access blocked: this app's request is invalid` | OAuth同意画面が未設定、または必要な情報が未入力 | 手順3を完了させる |
| テストユーザー以外のアカウントでログインできない | OAuth同意画面が「テスト」モードのまま | 手順3でテストユーザーに追加するか、個人利用を超える場合は「本番環境に公開」する（Googleの審査が必要になる場合あり） |
| ポップアップがブロックされる | ブラウザのポップアップブロック機能 | ブラウザのアドレスバー付近に出るブロック通知からこのサイトを許可する |

## 補足: 保存先について

現在の実装は Google Drive の「アプリデータフォルダ (`appDataFolder`)」にJSON形式で保存しています。これはユーザー本人のDrive容量を使いますが、**通常のDrive画面には表示されない（アプリ専用の隠し領域）** 仕様です。

もし「マイドライブ」に見える形で保存したい場合は、スコープを `https://www.googleapis.com/auth/drive.file` に変更し、`parents: ['appDataFolder']` の指定を外す必要があります。ご希望があればお知らせください。
